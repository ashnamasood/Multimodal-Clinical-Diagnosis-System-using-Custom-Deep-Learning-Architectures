How it works

        
Model and training details

      

      

        
01
Chest X-ray model
ResNet50-based multi-label classifier trained on NIH ChestX-ray14 with focal loss and per-class thresholds.

        
02
Skin and heart models
The API can integrate optional skin lesion and cardiac models when checkpoints are available.

        
03
Fusion logic
The backend merges modality outputs into a single risk score and a readable recommendation.

      

      

        
Training setup: CPU-optimized DataLoader parallelism, focal loss for imbalance, and validation-based threshold tuning.

        
Inference flow: upload → preprocess → model inference → confidence ranking → fused clinical summary.

        
Limitations: the chest X-ray model is still a research model and should not replace expert medical judgment.

      

    


    

      

        
FAQ

        
Common questions

      

      

        What does the system do?
        
It combines available modality outputs and presents a single review dashboard for clinical support.

      

      

        Can I use only the chest X-ray model?
        
Yes. The interface gracefully handles missing skin or heart inputs and still returns chest X-ray results.

      

      

        Why are some models unavailable?
        
Skin and heart models are optional. If checkpoints are missing, the app still works with the available X-ray model.

      

      

        Is this a medical device?
        
No. This demo is for research and educational use and should not replace professional medical judgment.
